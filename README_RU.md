# Trails From Zero Toolkit

> 🇬🇧 In English: [README.md](README.md)

Инструменты, оставшиеся после перевода *The Legend of Heroes: Trails
from Zero* (PC порт) на русский. Здесь всё, что нужно, чтобы засунуть
в игру свой алфавит и собрать рабочий билд — тот же тулчейн, что
использует русская локализация.

## Тестировалось на

- **Trails from Zero — Steam, версия 1.4.13.** Все инструменты в репо
  разрабатывались и round-trip-тестировались на этой сборке.
- **Trails to Azure** — тот же движок, те же форматы данных.
  Декомпиляторы из [`dt-tools/`](dt-tools/README.md) и сборщик
  [`font/`](font/README.md) применимы напрямую. Пять бинарных патчей в
  [`binpatch/`](binpatch/README.md) сделаны конкретно под `zero.exe` и под
  Azure пока не портированы.

На других PC-ревизиях *Zero* офсеты патчей могут отличаться. Java-
скрипты в [`binpatch/research/`](binpatch/research/) воспроизводят
анализ, если нужно найти эквиваленты в другом бинаре.

## Через что мы прошли

PC порт официально поддерживает японский, английский и встроенный
русский шрифт — поэтому очевидная отправная точка для нас это просто подменить
строки в существующих файлах. Для английского такой подход работает
(нормальный шрифт, 1-байтный ASCII). Для русского в коробке — нет:
встроенные глифы прижаты как кана, layout вылезает, и в нескольких
полях упираешься в жёсткие лимиты длины. Замена шрифта на нормальный
TTF чинит вид, но не длины. Чтобы починить длины — кодируем каждую
букву **одним байтом** (в 1-байтной halfwidth-зоне SJIS, которую
движок и так понимает) — это работает почти везде, кроме пяти
текстовых форматтеров, которые молча выкидывают байты выше `0x7F`.
Эти пять мест мы и пропатчили в `zero.exe`. Toolkit — это сборщик
шрифта, перекодировщик `._dt` и пять бинарных патчей.

## Какой путь ваш

Выбирайте один — он определяется языком, на который переводите, а не
вашим желанием:

- **Путь A — `passthrough`.** Для языков, которые в игре уже нормально
  работают (японский, английский) или для правки строк в стоковом
  русском билде. Буквы кодируются как стандартный 2-байтный SJIS или
  1-байтный ASCII — точно как в оригинальных файлах. Нужен только
  [`dt-tools/`](dt-tools/README.md) в режиме `--passthrough`. **Этот путь не
  сработает** для любого языка без нормального шрифта в игре, включая
  русский в коробке — упрётесь в проблемы layout / длины /
  читаемости, ради решения которых и сделан путь B.

- **Путь B — `halfwidth`.** Для любого языка, у которого в игре нет
  нормального шрифта (русский, украинский, греческий, иврит, …).
  Полный pipeline обязателен; ничто из этого не опционально. Нужны
  все три компонента:
  1. [`font/`](font/README.md) — собрать кастомный `font.itf` под ваш
     алфавит.
  2. [`dt-tools/`](dt-tools/README.md) в режиме `--halfwidth` —
     перекодировать каждый переведённый `._dt` так, чтобы буква
     занимала 1 байт.
  3. [`binpatch/`](binpatch/README.md) — применить пять surgical-патчей к
     `zero.exe`, чтобы движок перестал терять halfwidth-байты.

  Если пропустить любой из трёх — билд будет ломаться неочевидно:
  пропавшие буквы, случайные обрезки, бесконечные циклы на описаниях
  предметов.

## Что внутри

| Папка | Путь A | Путь B | Что делает |
|-------|:---:|:---:|------------|
| [`dt-tools/`](dt-tools/README.md) | да | да | Декомпиляторы / компиляторы `._dt` таблиц. Пошаговое использование, два режима кодировки, drag-and-drop и таблица замен — в README модуля. |
| [`font/`](font/README.md) | опционально | **обязательно** | Сборщик `font.itf`. Как собрать, как переориентировать под другой алфавит, какой игровой файл нужен — в README модуля. |
| [`binpatch/`](binpatch/README.md) | нет | **обязательно** | Пять микро-патчей к `zero.exe` для halfwidth-режима. Применение / откат и таблица патчей — в README модуля; ASM-уровень — в [`PATCHING_NOTES.md`](binpatch/PATCHING_NOTES.md). |

Пути в игре:

- `._dt` таблицы данных → `<игра>\data\text_us\`
- Шрифт → `<игра>\system_us\fontdat\font.itf`
- Сценарии (отдельный модуль, его пока нет в этом репо) →
  `<игра>\data\scena_us\`

Что нужно установить: Python 3.6+, `pip install freetype-py` (только
для сборщика шрифта), копия игры и TTF-шрифт с глифами вашего
алфавита. Перед началом обязательно сохраните копии `zero.exe`,
`system_us\fontdat\font.itf` и тех `._dt`, которые собираетесь
править.

## Авторство

- **Edrahor** — toolkit, упаковка, `byte_codec`, компиляторы `._dt`
  (книги, предметы, мемо, магазины, города), реверс и бинарные патчи,
  документация.
- **Ivdos** — оригинальный сборщик `font.itf`
  ([`font/glyph.py`](font/glyph.py)) и оригинальный декомпилятор
  квестов ([`dt-tools/dt_quest.py`](dt-tools/dt_quest.py)). Оба
  адаптированы для этого toolkit.
